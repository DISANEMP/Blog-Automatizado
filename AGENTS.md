# AGENTS.md

## Nome do projeto

Motor Afiliado Tech IA

## Objetivo principal

Criar um sistema web para gerar paginas, artigos e comparativos de produtos tecnologicos com foco em monetizacao por afiliado.

O projeto deve priorizar:

- artigos SEO;
- comparativos de produtos;
- paginas de decisao de compra;
- imagens e composicoes visuais proprias;
- links de afiliado;
- ads leves e pouco invasivos;
- experiencia limpa para o usuario;
- conteudo util, confiavel e focado em compra.

O foco nao e criar conteudo para Reels, TikTok, Shorts ou redes sociais. Nao incluir modulos de video curto, calendario de redes sociais ou automacao de postagem social nesta fase.

## Visao de negocio

O sistema deve ajudar a transformar produtos de tecnologia em conteudos de decisao de compra.

Exemplos de temas:

- Galaxy S24 FE vale a pena em 2026?
- S24 FE vs iPhone 15: qual comprar?
- Melhor celular para tirar foto de produto
- Melhor TV 55 polegadas ate R$2.000
- Samsung Crystal ou LG UHD: qual TV comprar?
- Notebook bom para professor em 2026
- Celular bom para gravar, trabalhar e vender online

O conteudo deve atrair pessoas que ja estao proximas da decisao de compra.

## Modelo de monetizacao

A prioridade e monetizacao por afiliado.

Ads podem existir, mas devem ser discretos, leves e nao poluir a experiencia.

Regras para ads:

- nao exagerar em banners;
- nao usar pop-up agressivo;
- nao prejudicar leitura;
- nao esconder conteudo;
- nao atrapalhar navegacao mobile;
- priorizar no maximo poucos espacos de anuncios por pagina;
- o foco principal da conversao deve ser o link de afiliado.

## Publico-alvo

Pessoas pesquisando antes de comprar produtos tecnologicos, especialmente:

- celulares;
- TVs;
- notebooks;
- fones;
- eletronicos uteis para trabalho;
- produtos para criadores de conteudo;
- equipamentos para quem vende online;
- tecnologia custo-beneficio.

## Posicionamento editorial

O site deve parecer um consultor de compra, nao um catalogo de ofertas.

Tom de voz:

- direto;
- confiavel;
- pratico;
- honesto;
- sem promessa exagerada;
- sem linguagem sensacionalista;
- sem parecer spam;
- com veredito claro.

O conteudo deve responder:

- vale a pena?
- para quem serve?
- para quem nao serve?
- qual preco comeca a compensar?
- quais concorrentes comparar?
- quais pontos fortes?
- quais limitacoes?
- qual produto comprar de acordo com o uso?

## Estrutura de paginas principais

O sistema deve futuramente permitir gerar:

1. Pagina de review individual
   Exemplo: "Galaxy S24 FE vale a pena em 2026?"

2. Pagina comparativa
   Exemplo: "S24 FE vs iPhone 15: qual comprar?"

3. Pagina ranking/guia
   Exemplo: "Melhores celulares para tirar foto de produto"

4. Pagina de categoria
   Exemplo: "Celulares custo-beneficio"

5. Pagina de ofertas selecionadas
   Exemplo: "Ofertas tech que fazem sentido hoje"

## Estrutura ideal de artigo SEO

Cada artigo deve seguir uma estrutura parecida com:

1. H1 com palavra-chave principal
2. Resposta curta logo no inicio
3. Veredito rapido
4. Para quem vale a pena
5. Para quem nao vale a pena
6. Principais pontos positivos
7. Principais pontos negativos
8. Comparativo com concorrentes
9. Faixa de preco ideal
10. Link de afiliado contextual
11. Aviso claro de link de afiliado
12. Perguntas frequentes
13. Conclusao com recomendacao honesta

Ao criar qualquer funcionalidade relacionada a conteudo, considerar:

- titulo SEO;

## Imagens e direitos autorais

Este projeto deve evitar uso indevido de imagens protegidas por direitos autorais.

Regras:

- nao baixar imagens aleatorias do Google;
- nao copiar imagem de review, blog, YouTube ou loja sem permissao;
- preferir imagens oficiais permitidas por programas de afiliado, quando houver;
- quando houver duvida de direito de uso, nao usar a imagem diretamente;
- criar composicoes proprias, mockups, cards visuais e criativos autorais;
- usar o modelo/produto como referencia textual e gerar briefing para criativo proprio;
- permitir campo para o usuario inserir imagem autorizada manualmente;
- permitir gerar um prompt de imagem ou briefing para designer/IA de imagem.

O sistema deve trabalhar com a ideia de "criativo proprio", nao simplesmente scraping de imagens.

## Comparativos visuais

O projeto deve priorizar comparativos com imagens/cards.

Exemplos de modulos visuais:

- Produto A vs Produto B;
- melhor para camera;
- melhor para bateria;
- melhor para preco;
- melhor para jogos;
- melhor para foto de produto;
- melhor para uso comum;
- tabela de pros e contras;
- card de veredito final;
- faixa de preco ideal.

Os comparativos devem ser limpos, premium, responsivos e faceis de ler no celular.

## Link de afiliado

Na primeira versao, o link de afiliado deve ser inserido manualmente pelo usuario.

Campos previstos:

- link original;
- link de afiliado;
- marketplace;
- preco atual;
- comissao estimada;
- observacao;
- data da ultima checagem.

Nao automatizar geracao de link de afiliado nesta fase, pois cada plataforma tem regras proprias.

## Aviso de afiliado

Toda pagina com link de afiliado deve ter aviso claro, por exemplo:

"Este conteudo contem link de afiliado. Podemos receber comissao se voce comprar pelo link, sem custo adicional para voce."

Esse aviso deve aparecer de forma discreta, mas visivel.

## Regras de qualidade

Nunca gerar conteudo enganoso.

Evitar:

- "melhor do mercado" sem justificativa;
- promessa exagerada;
- falsa urgencia;
- preco inventado;
- ficha tecnica nao confirmada;
- afirmacoes sem base;
- recomendacao de produto ruim so por comissao;
- excesso de links.

Priorizar:

- utilidade real;
- comparacao honesta;
- clareza;
- decisao de compra;
- confianca;
- SEO limpo;
- experiencia mobile.

## Regras para o Codex

Ao desenvolver este projeto:

- sempre preservar o foco em blog SEO + afiliado;
- nao adicionar TikTok, Reels ou Shorts sem solicitacao explicita;
- nao criar publicacao automatica em massa;
- nao usar scraping agressivo;
- nao baixar imagens sem permissao;
- sempre pensar em revisao humana antes da publicacao;
- manter codigo limpo, modular e escalavel;
- priorizar performance e SEO;
- escrever comentarios uteis quando necessario;
- evitar dependencias desnecessarias;
- criar componentes reutilizaveis;
- manter layout responsivo;
- proteger chaves de API em variaveis de ambiente;
- nunca expor secrets no frontend;
- usar nomes claros para arquivos, funcoes e componentes.

2. criar comparativos claros;
3. melhorar chance de ranqueamento no Google;
4. aumentar cliques em links afiliados;
5. manter experiencia limpa;
6. evitar spam;
7. respeitar direitos de imagem;
